# ClaudeVibeFlow: 상세 파일 분석 보고서

로컬 워크스페이스 내의 `claude-vibe-flow` 디렉토리를 깊이 있게 분석한 결과, 이는 **Claude Code**(CLI)를 활용한 자율적 에이전트 생태계를 구축하기 위한 고도로 정교한 프레임워크인 것으로 확인되었습니다.

## 1. 전반적인 아키텍처

이 프로젝트는 단순히 코딩 스타일을 제안하는 것을 넘어, 개발의 전 과정을 자동화하는 **에이전트 오케스트레이션(Orchestration)** 시스템을 지향합니다.

### 핵심 구조:
- **`agents/` (15개)**: 특정 도메인(Git, 리뷰, 설계, 테스트 등)에 특화된 전문 에이전트들입니다.
- **`commands/`**: 워크플로우를 자동 실행하는 슬래시 명령어(`new-feature`, `check-setup`)를 제공합니다.
- **`config/`**: 사용자의 의도(Intent)를 분석하여 적절한 에이전트로 연결하는 라우팅 규칙(`intent-routing.md`)을 담고 있습니다.

## 2. 주요 에이전트 분석

| 에이전트 그룹 | 주요 에이전트 | 핵심 역할 |
| :--- | :--- | :--- |
| **Orchestration** | `pm-orchestrator` | 복잡도/명확도를 분석하여 단일 에이전트 또는 파이프라인으로 업무를 라우팅함. |
| **Critical** | `git-guardian` | **Vibe Coding** 스타일의 브랜치 네이밍(`vibe/*`) 및 원자적 커밋 자동화. |
| **Critical** | `issue-fixer` | 에러 발생 시 디버깅 및 수정을 전문으로 담당. |
| **Quality** | `code-reviewer` | 변경 사항 발생 후 자동으로 코드 품질 및 보안 리뷰 수행. |
| **Meta** | `agent-manager` | 에이전트 생태계의 상태를 확인하고 새로운 에이전트 생성을 관리. |

## 3. 워크플로우 및 자동화 (Vibe Coding 최적화)

이 템플릿의 가장 큰 특징은 **자동 트리거(Automatic Trigger)** 시스템입니다.

- **브랜치 관리**: 세션이 시작되면 `git-guardian`이 자동으로 현재 브랜치를 확인하고 필요시 `vibe/[context]-[feature]` 형식의 새 브랜치를 생성합니다.
- **의도 기반 라우팅**: 사용자가 "타입 검증해줘"라고 하면 `intent-routing.md` 규칙에 따라 `type-sync-checker`가 자동으로 소환됩니다.
- **파이프라인 실행**: 복잡한 요청(`Epic` 레벨)의 경우 `planner` → `architect` → `spec-validator` → `vibe-implementer` 순서로 이어지는 정교한 작업 파이프라인을 구성합니다.

## 4. 커스터마이징 및 확장성

- **에이전트 추가**: `agents/` 디렉토리에 새 에이전트 파일을 추가하여 프로젝트 특화 에이전트를 쉽게 생성할 수 있습니다.
- **`CLAUDE.md` 통합**: 프로젝트별로 `intent-routing.md`를 오버라이드하거나 트리거 테이블을 정의하여 동작을 맞춤 설정할 수 있습니다.

## 5. 결론 및 권장 사항

이 템플릿은 단순한 텍스트 파일 모음이 아니라, Claude를 실제 **자율 개발팀**처럼 활용할 수 있게 해주는 "운용 체계"입니다. 

- **추천**: 새 기능을 개발할 때는 `/project:new-feature "기능명"` 명령어를 사용하여 시스템이 제안하는 표준 워크플로우를 따라보시기 바랍니다.
- **주의**: 모든 프롬프트에 컨텍스트가 포함되므로 `CLAUDE.md`를 300줄 이내로 유지하여 토큰 효율성을 관리하는 것이 중요합니다.
